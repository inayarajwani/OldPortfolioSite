Portfolio Website Check-in Checklist

Your site should meet the following requirements:

  • Styling is consistent across pages.

  • Links all work, including links to external CSS
  • You have at least three pages.
  • You have a navbar containing links between all pages
  • Assets are in place and look good on your site.
  • Spacing between elements is consistent and as-intended
  • You have a header indicating what the site is about (E.g. Sarah’s Tech Portfolio)
  • You have a footer section with useful links.
    If linking to social media, be careful what information you disclose.
    Avoid links to media containing personal details
  • You have a homepage with samples from your portfolio (or, for now, placeholder images that say “Coming Soon”)
  • You have an “About Me” page that’s informative and appealing. It should be something like a resume

Error Types

Answer the following questions in a word document. You may wish to screen-cap your code for questions 2, 3, and 4, and paste it into the document. The code should be your own.

  1. Define each of the following types of coding errors.

    • Syntax error: A character or string incorrectly placed in a command or instructions that causes a failure in execution.
    • Runtime error: A program error that occurs while the program is running.
    • Logic error: A bug in a program that causes it to operate incorrectly, but no to terminate abnormally or crash.
      It is a mistake in a program’s source code that results in incorrect or unexpected behavior.

  2. Write three examples of code that contain a syntax error. Use a different type of syntax error for each.

    • var myname( );
    • var firstName = ’Inaya‘;
    • var favoriteFodo = “Burrito”;

  3. Write two examples of code that contain a runtime error. Use a different type of runtime error for each.

    • var myName = “user”;
      console.log(person);
    • var favoriteFood=Burrito;
      var myName= “Inaya”;
      console.log(favoriteFood);
      console.log(myFirstName);

  4. Write two examples of code that contain a logic error. Use a different error for each.

    • if (‘solution’ == 5);
    • if (solution == 5) {
      answer = ‘correct’;
      }
      else (solution == 4) {
      answer = ‘incorrect’;
      }
      }

  5. Explain how you would find each type of error (debug) in a program.

    • Ask friend to skim over code
    • Run over code to look for:
      • Missing ";"
      • ” “ or ’‘ (backwards quotation marks)
      • Uppercase or lowercase letters in the wrong places
      • ( ), { }, < > - missing open or closing bracket
    • Check the code in the console log

    Debugging runtime errors:

    • Identify that there is an error by knowing if the program:
      • Takes too long to operate
      • Runs but crashes while running

    Debugging logic errors:

    • Run debugging codes
    • Run the code through your head to ensure it makes sense
